## 版本命名规范

版本号格式：`主版本号.次版本号.修订号`

- **主版本号**: 重大架构变更或功能重构
- **次版本号**: 新功能添加或重要优化  
- **修订号**: Bug修复和性能优化

## 提交规范

提交信息应遵循以下格式：
```
类型(范围): 简短描述

详细描述（可选）
```

### 提交类型
- `feat`: 新功能
- `fix`: Bug修复  
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建过程或辅助工具变动

### 示例
```
feat(discovery): 新增UP主自动发现功能

- 实现基于搜索的UP主发现算法
- 添加防封策略和随机延迟
- 支持多页面深度采集
```
